BAB I
PENDAHULUAN

1.1 Latar Belakang

PT Terminal Petikemas Surabaya (TPS) merupakan operator terminal petikemas kelas dunia yang berlokasi di Pelabuhan Tanjung Perak, Surabaya. Sebagai perusahaan yang terus berkembang dan bertransformasi digital, TPS memiliki kebutuhan akan sistem pengelolaan konten website (Content Management System / CMS) yang handal, aman, dan mudah dikelola.

Sistem CMS website TPS yang baru dikembangkan membutuhkan infrastruktur database yang terpisah dari application server untuk memastikan performa optimal, keamanan data, serta kemudahan dalam pengelolaan dan pemeliharaan. Pemisahan ini mengikuti praktik terbaik (best practice) arsitektur multi-tier yang memisahkan presentation layer, application layer, dan data layer.

Dalam rangka implementasi sistem tersebut, PT PRIMALINK selaku mitra teknologi ditugaskan untuk melakukan instalasi, konfigurasi, dan migrasi database PostgreSQL pada server yang telah disediakan oleh TPS di lingkungan jaringan internal perusahaan.

1.2 Tujuan

Dokumen ini disusun dengan tujuan sebagai berikut:

  1. Mendokumentasikan seluruh proses instalasi dan konfigurasi PostgreSQL database server pada infrastruktur TPS.
  2. Mendokumentasikan proses migrasi (restore) data dari database existing ke server baru.
  3. Mendokumentasikan konfigurasi keamanan yang diterapkan pada database server.
  4. Memberikan panduan teknis untuk pemeliharaan dan troubleshooting di masa mendatang.
  5. Sebagai laporan progres pekerjaan kepada PT Terminal Petikemas Surabaya.

1.3 Ruang Lingkup

Ruang lingkup pekerjaan yang didokumentasikan dalam laporan ini meliputi:

  1. Instalasi PostgreSQL 16 pada server database (172.19.154.92).
  2. Konfigurasi koneksi, autentikasi, dan keamanan database.
  3. Pembuatan user dan database untuk sistem CMS.
  4. Restore (migrasi) data dari backup database production.
  5. Integrasi dan pengujian koneksi dengan application server (172.19.154.93).
  6. Verifikasi dan validasi seluruh data hasil migrasi.

BAB III
INSTALASI POSTGRESQL

3.1 Persiapan Sistem Operasi

Sebelum melakukan instalasi PostgreSQL, dilakukan persiapan pada sistem operasi server database (172.19.154.92) yang meliputi update paket sistem dan instalasi dependensi yang dibutuhkan.

1 Update Sistem Operasi 
sudo apt update && sudo apt upgrade -y

Perintah ini memperbarui daftar paket dan menginstall seluruh update keamanan yang tersedia untuk Ubuntu 24.04 LTS.

2 Instalasi Dependensi 
sudo apt install -y wget curl gnupg2 lsb-release

Paket-paket ini diperlukan untuk mengunduh dan memverifikasi repository PostgreSQL.

3.2 Instalasi PostgreSQL 16

PostgreSQL versi 16 dipilih karena merupakan versi stabil terbaru yang tersedia di repository resmi Ubuntu 24.04 LTS, dengan dukungan jangka panjang dan fitur-fitur terbaru yang dibutuhkan oleh sistem CMS.

3 Instalasi PostgreSQL dari Repository Ubuntu 
sudo apt install -y postgresql postgresql-contrib

Paket postgresql berisi server database, sedangkan postgresql-contrib berisi ekstensi tambahan termasuk uuid-ossp yang dibutuhkan untuk generate UUID.

4 Mengaktifkan Autostart Service 
sudo systemctl enable postgresql
sudo systemctl start postgresql

Perintah ini memastikan PostgreSQL otomatis berjalan ketika server di-restart.

3.3 Verifikasi Instalasi

BAB IV
KONFIGURASI DATABASE

4.1 Konfigurasi Koneksi (postgresql.conf)

File konfigurasi utama PostgreSQL perlu diubah agar server database dapat menerima koneksi dari application server. Secara default, PostgreSQL hanya menerima koneksi dari localhost.

File: /etc/postgresql/16/main/postgresql.conf

1 Mengubah Listen Address 
# Sebelum (default - hanya localhost)
listen_addresses = 'localhost'

# Sesudah (menerima koneksi dari semua interface)
listen_addresses = '*'

Parameter listen_addresses = '*' memungkinkan PostgreSQL menerima koneksi TCP dari network interface manapun. Keamanan tetap dijaga melalui konfigurasi pg_hba.conf yang membatasi IP yang diizinkan.

Tabel 4.1 — Parameter Konfigurasi PostgreSQL
ParameterNilaiKeterangan
listen_addresses'*'Menerima koneksi dari semua interface
port5432Port standar PostgreSQL
max_connections100Maksimum koneksi bersamaan
shared_buffers128MBMemory untuk shared buffer

4.2 Konfigurasi Autentikasi (pg_hba.conf)

File pg_hba.conf (Host-Based Authentication) mengontrol siapa yang boleh terhubung ke database, dari mana, dan dengan metode autentikasi apa. Konfigurasi ini merupakan lapisan keamanan utama pada level database.

File: /etc/postgresql/16/main/pg_hba.conf

2 Menambahkan Akses untuk Application Server 
# Koneksi lokal (default)
local   all   postgres                      peer
local   all   all                           peer
host    all   all       127.0.0.1/32        scram-sha-256
host    all   all       ::1/128             scram-sha-256

# Tambahan: akses dari Application Server
host    all   all       172.19.154.93/32    scram-sha-256
Penjelasan Konfigurasi:
• host — Koneksi melalui TCP/IP
• all all — Berlaku untuk semua database dan semua user
• 172.19.154.93/32 — Hanya IP application server yang diizinkan (subnet mask /32 = exact match)
• scram-sha-256 — Metode autentikasi terenkripsi (paling aman yang tersedia)

4.3 Konfigurasi User dan Password

Untuk keamanan, password user postgres (superuser) di-set dengan password yang kuat menggunakan metode autentikasi scram-sha-256.

3 Set Password User postgres 
sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD 'TpsDB2026Secure';"
Catatan Keamanan: Password yang tercantum dalam dokumen ini bersifat rahasia. Distribusi dokumen ini harus dibatasi kepada personel yang berwenang saja. Disarankan untuk mengganti password secara berkala sesuai kebijakan keamanan TPS.

4.4 Pembuatan Database

Database tps_web dibuat untuk menyimpan seluruh data sistem CMS website. Ekstensi uuid-ossp diaktifkan karena sistem CMS menggunakan UUID sebagai primary key.

4 Membuat Database dan Ekstensi 
sudo -u postgres psql

-- Membuat database
CREATE DATABASE tps_web;

-- Beralih ke database baru
\c tps_web

-- Mengaktifkan ekstensi UUID
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Verifikasi
\dx
# Output: uuid-ossp | 1.1 | public | generate universally unique identifiers

# Transfer backup file ke server database
scp tps_web_backup.sql root@172.19.154.92:/tmp/

Proses restore dilakukan menggunakan tool psql yang membaca file SQL dump dan menjalankan seluruh perintah SQL untuk membuat struktur tabel dan memasukkan data.

2 Restore Database dari Backup

sudo -u postgres psql tps_web < /tmp/tps_web_backup.sql

# Atau menggunakan pg_restore untuk format custom
# sudo -u postgres pg_restore -d tps_web /tmp/tps_web_backup.dump

6.2 Metode Autentikasi

Metode autentikasi yang digunakan adalah scram-sha-256, yang merupakan metode autentikasi paling aman yang tersedia di PostgreSQL 16. Metode ini menggunakan challenge-response mechanism dengan hashing SHA-256, sehingga password tidak pernah dikirimkan dalam bentuk plain text melalui jaringan.

Tabel 6.2 — Perbandingan Metode Autentikasi PostgreSQL
MetodeKeamananDigunakan
trustTidak aman — tanpa password❌
passwordRendah — plain text❌
md5Sedang — hash MD5❌
scram-sha-256Tinggi — hash SHA-256✅

6.3 Firewall (UFW)

Sebagai lapisan keamanan tambahan, disarankan untuk mengkonfigurasi firewall menggunakan UFW (Uncomplicated Firewall) pada server database untuk membatasi port yang dapat diakses. 

# Konfigurasi firewall (rekomendasi)
sudo ufw allow from 172.19.154.93 to any port 5432
sudo ufw allow ssh
sudo ufw enable

BAB VII
INTEGRASI DENGAN APPLICATION SERVER

7.1 Konfigurasi Connection String

Application server (172.19.154.93) menjalankan aplikasi CMS dalam container Docker. Koneksi ke database dikonfigurasi melalui environment variable DATABASE_URL dalam format connection string PostgreSQL. 

# Format Connection String
postgresql://[user]:[password]@[host]:[port]/[database]

# Konfigurasi aktual
DATABASE_URL="postgresql://postgres:TpsDB2026Secure@172.19.154.92:5432/tps_web"

Connection string ini dikonfigurasi pada dua tempat:

  1. Docker Compose (docker-compose.yml) — sebagai environment variable untuk container.
  2. Prisma ORM (app/db/.env) — untuk ORM database yang digunakan oleh aplikasi.

7.2 Prisma ORM

Aplikasi CMS menggunakan Prisma ORM versi 5.11.0 sebagai database abstraction layer. Prisma menyediakan type-safe query builder dan auto-generated client berdasarkan schema database.

File schema: app/db/prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

BAB IX
KESIMPULAN DAN REKOMENDASI

9.1 Kesimpulan

Berdasarkan seluruh proses yang telah dilakukan, dapat disimpulkan bahwa:

  1. Instalasi PostgreSQL 16 pada server database (172.19.154.92) telah berhasil dilakukan dengan status service active dan berjalan normal pada port 5432.
  2. Konfigurasi keamanan telah diterapkan dengan baik, meliputi pembatasan akses IP hanya dari application server (172.19.154.93) dan penggunaan metode autentikasi scram-sha-256.
  3. Migrasi (restore) database dari backup production telah berhasil dilakukan. Seluruh 16 tabel dengan total 17.773 record konten telah berhasil dimigrasikan dan terverifikasi.
  4. Integrasi antara application server dan database server telah berfungsi dengan baik, dibuktikan dengan seluruh halaman website yang dapat diakses dan menampilkan data dengan benar.
  5. Pengujian end-to-end menunjukkan seluruh komponen sistem berfungsi normal, termasuk homepage, halaman berita, halaman download dokumen, dan panel admin CMS.

9.2 Rekomendasi

Untuk menjaga keberlangsungan dan keamanan sistem, direkomendasikan hal-hal berikut:

  1. Backup Berkala — Mengimplementasikan backup otomatis database menggunakan pg_dump dengan jadwal harian, dan menyimpan backup di lokasi terpisah. 
    pg_dump -h 172.19.154.92 -U postgres tps_web > backup_$(date +%Y%m%d).sql
  2. Monitoring — Memasang monitoring untuk memantau kesehatan database (koneksi aktif, disk usage, query performance).
  3. Rotasi Password — Mengganti password database secara berkala (minimal setiap 90 hari) sesuai kebijakan keamanan perusahaan.
