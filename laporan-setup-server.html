BAB I
PENDAHULUAN

1.1 Latar Belakang

PT Terminal Petikemas Surabaya (TPS) sebagai salah satu terminal petikemas terbesar di Indonesia memiliki kebutuhan infrastruktur teknologi informasi yang handal dan scalable. Website korporat TPS merupakan salah satu kanal komunikasi penting yang menyajikan informasi layanan, berita, dan berbagai informasi publik kepada stakeholder.

Dalam rangka meningkatkan performa, keamanan, dan kemudahan pengelolaan sistem, dilakukan migrasi infrastruktur website dari server lama ke infrastruktur baru dengan arsitektur yang lebih modern. Migrasi ini mencakup pemisahan server database dan aplikasi, serta implementasi containerization menggunakan Docker untuk memudahkan deployment dan maintenance.

Dokumen ini disusun sebagai laporan teknis yang mendokumentasikan proses setup server web dan database, termasuk langkah-langkah implementasi, konfigurasi, dan hasil pengujian sistem.

1.2 Tujuan

Tujuan dari kegiatan setup server ini adalah:

  1. Membangun infrastruktur server yang terpisah antara database dan aplikasi untuk meningkatkan keamanan dan performa.
  2. Mengimplementasikan containerization menggunakan Docker untuk memudahkan deployment dan scaling aplikasi.
  3. Melakukan migrasi data dari server lama ke server baru tanpa kehilangan data.
  4. Memastikan kontinuitas layanan website TPS selama dan setelah proses migrasi.
  5. Menyediakan dokumentasi teknis yang lengkap untuk keperluan maintenance di masa mendatang.

1.3 Ruang Lingkup

Ruang lingkup pekerjaan dalam dokumen ini meliputi:

  1. Setup dan konfigurasi server database PostgreSQL 16 pada server dedicated.
  2. Setup dan konfigurasi server aplikasi dengan Docker pada server terpisah.
  3. Migrasi database dari server development ke server production baru.
  4. Migrasi file uploads dari server production lama ke server production baru.
  5. Konfigurasi jaringan dan keamanan antar server.
  6. Pengujian fungsional aplikasi setelah deployment.

BAB III
IMPLEMENTASI

3.1 Setup Server Database

Server database dikonfigurasi menggunakan PostgreSQL 16 yang diinstall secara native (standalone) pada sistem operasi Ubuntu. Pemilihan instalasi native dibandingkan container bertujuan untuk mendapatkan performa optimal dan kemudahan backup.

3.1.1 Instalasi PostgreSQL

Perintah instalasi PostgreSQL pada server database:

apt-get update apt-get install -y postgresql postgresql-contrib systemctl enable postgresql systemctl start postgresql

3.1.2 Konfigurasi Keamanan

Konfigurasi keamanan PostgreSQL dilakukan dengan mengatur password untuk user postgres dan membatasi akses hanya dari server aplikasi.

# Set password untuk user postgres sudo -u postgres psql -c "ALTER USER postgres WITH PASSWORD 'TpsDB2026Secure';" # Konfigurasi pg_hba.conf untuk akses remote echo "host all all 172.19.154.93/32 scram-sha-256" >> /etc/postgresql/16/main/pg_hba.conf # Konfigurasi postgresql.conf untuk listen address echo "listen_addresses = '*'" >> /etc/postgresql/16/main/postgresql.conf # Restart PostgreSQL systemctl restart postgresql

3.1.3 Pembuatan Database

Database untuk aplikasi dibuat dengan perintah:

sudo -u postgres psql -c "CREATE DATABASE tps_web;"

3.2 Setup Server Aplikasi

Server aplikasi dikonfigurasi menggunakan Docker untuk menjalankan aplikasi dalam container. Hal ini memudahkan deployment, scaling, dan maintenance aplikasi.

3.2.1 Instalasi Docker

Instalasi Docker menggunakan repository resmi Docker:

# Install dependencies apt-get update apt-get install -y ca-certificates curl # Add Docker GPG key install -m 0755 -d /etc/apt/keyrings curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc chmod a+r /etc/apt/keyrings/docker.asc # Add Docker repository echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \ https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \ tee /etc/apt/sources.list.d/docker.list > /dev/null # Install Docker apt-get update apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

3.2.2 Struktur Direktori

Struktur direktori yang dibuat pada server aplikasi:

/opt/ ├── tps-app/ # Direktori aplikasi │ ├── Dockerfile # Docker build file │ ├── docker-compose.yml # Docker Compose configuration │ ├── app/ # Source code aplikasi │ │ ├── db/ # Prisma schema dan konfigurasi │ │ ├── srv/ # Server-side code │ │ └── web/ # Frontend assets │ └── pkgs/ # Package modules │ └── tps-data/ # Direktori data (external volume) ├── uploads/ # File uploads (4.7 GB) └── logs/ # Application logs

3.3 Migrasi Data

3.3.1 Migrasi Database

Data database dimigrasi dari server development (103.187.215.236:5333) ke server production baru. Proses migrasi menggunakan pg_dump untuk export dan psql untuk import.

# Export dari server development PGPASSWORD='[password]' pg_dump -h 103.187.215.236 -p 5333 -U postgres -d tps-web > backup_dev.sql # Import ke server production sudo -u postgres psql -d tps_web -f backup_dev.sql

3.3.2 Migrasi File Uploads

File uploads dimigrasi dari server production lama (172.19.154.97) ke server production baru. Total ukuran file yang dimigrasi adalah 4.7 GB.

# Compress files di server lama cd /var/www/html/tps-web/data tar -czvf /tmp/files_backup.tar.gz files/ # Transfer ke server baru scp /tmp/files_backup.tar.gz root@172.19.154.93:/opt/tps-data/ # Extract di server baru cd /opt/tps-data tar -xzvf files_backup.tar.gz mv files uploads

3.4 Konfigurasi Docker

3.4.1 Dockerfile

Dockerfile untuk build image aplikasi:

# TPS Web Application FROM oven/bun:1.1.20-debian # Install system dependencies RUN apt-get update && apt-get install -y \ curl \ git \ openssl \ && rm -rf /var/lib/apt/lists/* # Set working directory WORKDIR /app # Copy all application code COPY . ./ # Install dependencies RUN bun install --production # Create necessary directories RUN mkdir -p /app/uploads /app/logs # Generate Prisma client RUN cd app/db && rm -f .env && bunx prisma generate --schema=prisma/schema.prisma # Set environment variables ENV NODE_ENV=production ENV PORT=3300 # Expose port EXPOSE 3300 # Health check HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \ CMD curl -f http://localhost:3300/ || exit 1 # Start application CMD ["bun", "run", "prod"]

3.4.2 Docker Compose

Konfigurasi Docker Compose untuk orchestration:

services: tps-app: build: context: . dockerfile: Dockerfile container_name: tps-app restart: unless-stopped ports: - "3300:3300" environment: - NODE_ENV=production - DATABASE_URL=postgresql://postgres:TpsDB2026Secure@172.19.154.92:5432/tps_web - PORT=3300 volumes: - /opt/tps-data/uploads:/data/files - /opt/tps-data/logs:/app/logs - /opt/tps-app/app/db/.env:/app/app/db/.env:ro networks: - tps-network healthcheck: test: ["CMD", "curl", "-f", "http://localhost:3300/"] interval: 30s timeout: 10s retries: 3 start_period: 40s networks: tps-network: driver: bridge

3.4.3 Deployment

Perintah untuk build dan deploy aplikasi:

# Build image docker compose build # Start container docker compose up -d # Verifikasi status docker compose ps docker compose logs --tail=20

BAB V
PENUTUP

5.1 Kesimpulan

Berdasarkan implementasi dan pengujian yang telah dilakukan, dapat disimpulkan bahwa:

  1. Setup server database PostgreSQL 16 dan server aplikasi Docker telah berhasil dilakukan dengan arsitektur two-tier yang memisahkan database dan aplikasi.
  2. Migrasi data dari server development (database) dan server production lama (files) ke infrastruktur baru telah berhasil dengan total 17,728 content records dan 4.7 GB file uploads.
  3. Aplikasi website TPS berjalan pada infrastruktur baru dengan fitur-fitur publik berfungsi normal. Ditemukan bug pada fungsi simpan konten di panel admin yang perlu diperbaiki, serta beberapa fitur admin lainnya yang masih memerlukan verifikasi lebih lanjut.
  4. Implementasi Docker memudahkan proses deployment dan memberikan kemampuan untuk scaling di masa mendatang.
  5. Pemisahan server database dan aplikasi meningkatkan keamanan dan memudahkan maintenance masing-masing komponen.

5.2 Rekomendasi

Untuk pengembangan dan pemeliharaan sistem ke depan, direkomendasikan hal-hal berikut:

  1. Perbaikan Bug Admin - Segera perbaiki bug pada fungsi simpan konten di panel admin dan lakukan verifikasi menyeluruh terhadap fitur-fitur admin lainnya sebelum sistem digunakan untuk operasional.
  2. Setup Nginx Reverse Proxy - Implementasikan Nginx sebagai reverse proxy di depan aplikasi untuk load balancing, caching, dan SSL termination.
  3. SSL/TLS Certificate - Pasang sertifikat SSL untuk mengaktifkan HTTPS dan mengamankan komunikasi client-server.
  4. Automated Backup - Konfigurasi backup otomatis harian untuk database dan file uploads dengan retention policy yang sesuai.
  5. Monitoring - Implementasikan monitoring sistem menggunakan tools seperti Prometheus dan Grafana untuk memantau health server dan aplikasi.
  6. CI/CD Pipeline - Bangun pipeline CI/CD untuk otomatisasi testing dan deployment saat ada perubahan kode.
  7. Disaster Recovery Plan - Susun dan dokumentasikan prosedur disaster recovery untuk mengantisipasi kejadian tidak terduga.
  8. Security Hardening - Lakukan hardening keamanan server termasuk firewall configuration, fail2ban, dan security patches rutin.
