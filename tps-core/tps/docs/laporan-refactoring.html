Tujuan Tahapan Arsitektur Komponen Styling Hasil

1. Tujuan Refactoring

Membuat aplikasi TPS website berjalan mandiri (standalone) tanpa koneksi ke CMS Prasi eksternal (prasi.avolut.com).

Masalah Sebelumnya:

  • Frontend bergantung pada Prasi renderer (React bundle dari external)
  • Data jadwal kapal di-fetch melalui halaman yang sudah di-render
  • Instagram data di-fetch dari external source
  • Deployment download dari prasi.avolut.com/prod-zip

Target:

  • 100% independen dari Prasi CMS external
  • Data langsung dari database lokal dan WSDL internal
  • Custom SSR components yang maintainable
  • Performa lebih cepat (no React hydration overhead)

2. Tahapan Implementasi

1

Route /new dengan Prasi Renderer

Membuat route baru /new yang menggunakan Prasi renderer tetapi dengan data dari database lokal.

File: app/srv/api/_frontend.ts

  • Query content dari PostgreSQL (banner, profile, services, dll)
  • Render menggunakan Prasi bundled components
  • Data dikirim via window.___content
Selesai Data lokal, renderer masih Prasi
2

Data Layer Independen

Menghapus dependency external untuk data jadwal kapal dan Instagram.

Jadwal Kapal (WSDL)

File: app/srv/utils/jadwal-kapal.ts

URL: http://www.tps.co.id:9090/WSCBS/Vessel
Username: wsvessel
Password: wsvessel123

Instagram Data (Local)

File: app/srv/data/ig.json

Data Instagram di-cache lokal, tidak fetch dari external API.

Selesai Semua data dari sumber internal
3

Custom SSR Components

Membuat komponen SSR menggunakan template literals (tanpa React runtime). Route baru: /custom

Komponen yang dibuat:

Header.ts HeroBanner.ts Services.ts Profile.ts Throughput.ts VesselSchedule.ts News.ts Customers.ts Instagram.ts Footer.ts HomePage.ts html.ts
Selesai Full SSR tanpa Prasi renderer
4

Styling Match dengan Prasi

Menyesuaikan desain Header dan Footer agar sesuai dengan tampilan TPS asli.

Header

  • Desktop: 2-tier (white upper + blue menu)
  • Upper height: 85px
  • Container max-width: 1100px
  • Blue color: #0475BC

Footer

  • Background: #0475BC
  • Box shadow: inset 0 4px 4px rgba(0,0,0,.25)
  • Padding: 50px top, 30px bottom
  • 2 columns: Sitemap + Kontak
Selesai Desain sesuai TPS theme

3. Arsitektur

Perbandingan /new vs /custom

Aspek /new (Legacy) /custom (Standalone)
Rendering Client-side (React hydration) Server-side (SSR)
Renderer Prasi bundled components Custom SSR components
HTML Output ~42 baris (skeleton) ~1500 baris (full HTML)
JS Bundle Prasi React bundle (~200KB+) Tailwind CDN + minimal JS
SEO Requires JS Full SEO support
Dependencies Prasi bundle (app/web/deploy/) None (standalone)

4. Struktur File

tps-core/tps/app/srv/
├── api/
│   ├── _custom.ts        # API endpoint /custom (SSR homepage)
│   ├── _frontend.ts      # API endpoint /new (Prasi renderer)
│   └── _static.ts        # Static file serving
│
├── components/           # SSR Components (template literals)

│   ├── Header.ts         # Header + Navigation

│   ├── Services.ts       # Service cards grid
│   ├── Profile.ts        # Tentang TPS section
│   ├── Throughput.ts     # Statistics cards

├── utils/

│
